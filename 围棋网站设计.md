# 围棋自学网站 · 设计文档（v1 草案）

> 目标：做一个**完全免费、自己说了算**的围棋练习站。核心两件事——**跟 AI 对练**、**练死活题**。
> 原则：**纯静态、可自托管、无会员墙、能离线用**。

---

## 1. 产品定位与原则

| 原则 | 说明 |
| --- | --- |
| 零成本 | 不接任何付费墙。部署到 GitHub Pages / CloudStudio 等免费静态托管即可，自己想关就关、想搬就搬。 |
| 自托管 | 代码开源、数据在本地。不依赖任何第三方账号。 |
| 离线可用 | 做成 PWA，第一次打开后断网也能练（棋盘、AI、已下载题库都在本地）。 |
| 渐进式 | 初学者从 9/13 路 + 轻量 AI 入门，进阶再切 KataGo 强引擎、上 19 路。 |
| 数据归你 | 对局记录、做题进度、错题本都存在浏览器本地（IndexedDB），可导出。 |
| 视觉风格 | **中国水墨风**：宣纸米白底、焦墨黑、朱砂红点缀、青灰辅助，留白克制。 |
| 无账号 | 纯本地、不接账号体系；不做多端云同步（数据只在当前浏览器）。 |

**目标用户**：围棋初学者 → 业余级爱好者。不做赛事/社交/直播、不做人人双人对下。

---

## 2. 核心功能

### 2.1 AI 对练（初级 / 中级 / 高级 三档）

- **棋盘**：9 路 / 13 路 / 19 路，随时切换。
- **初级 · 轻量 AI（浏览器内纯 JS）**
  - 规则 + 启发式打分，弱但**即时落子、零下载**，适合纯入门和让子陪练。
  - 可设让子数（2/4/6…）、执黑/执白、AI 思考速度。
- **中级 · KataGo WASM（低~中强度，本地推理）**
  - 浏览器内 WASM 跑 KataGo，**仍在用户本机算**，不连服务器，保持"纯静态、免费"。
  - 适合有基础、想认真对练的玩家；首次加载模型较大（几十 MB），**懒加载**。
- **高级 · KataGo WASM（高强度，本地推理）**
  - 同一引擎、更高算力/搜索深度，接近强业余~职业级陪练。
  - 低端机可能偏慢，可限制线程数/算力。
- **对局基础能力**：落子 / 提子 / 悔棋 / 数子判定 / 计时 / SGF 导出 / 死活标记（叉号、圆圈、数字）。
- **模式说明**：只做「人 vs AI」，不做人人双人对下。

### 2.2 死活题学习

- **题库来源**：内置 **50 道**精选经典题起步 + 支持导入开源 SGF 死活题库（持续扩充）。
- **解题交互**：读题 → 试手 → **即时判定正误** → 错了给提示、对了展示正解分支。
- **组织方式**：难度分级（入门/初级/中级/高级）、类型标签（做活 / 杀棋 / 对杀 / 官子 / 手筋）。
- **进度与错题**：本地记录已做题、正确率、自动归集**错题本**，可针对性重练。
- **题面渲染**：基于 SGF 解析，支持多分支正解展示。

## 3. 架构设计（纯静态，无后端）

整个站点就是一组静态文件，浏览器里跑全部逻辑：

```
浏览器 (单页应用 SPA)
├─ 棋盘渲染层   (SVG，清晰、易点选)
├─ 规则引擎     (落子合法性 / 提子 / 打劫 / 数子)
├─ AI 层
│   ├─ 轻量 AI       (纯 JS 启发式)
│   └─ KataGo WASM   (懒加载，本地推理)
├─ 死活题引擎   (SGF 解析 + 题目交互 + 判定)
└─ 本地存储     (IndexedDB：对局 / 进度 / 错题本)
```

**为什么能纯静态又用强引擎**：KataGo 以 WASM 形式在用户浏览器本地运行，算力来自用户自己的设备，不依赖服务器，因此依然"零成本、免会员"。

**技术栈建议（保持轻量）**
- 语言/框架：原生 ES Modules，或轻量框架 **Preact**（体积小、够用）。不引重型框架。
- 构建：**Vite**（开发快、产物小）。
- 棋盘：**SVG** 绘制（线条、星位、棋子都是矢量，缩放清晰、点击命中容易）。
- 棋谱：**自写或复用开源 SGF 解析**（SGF 是围棋通用格式）。
- 存储：**IndexedDB**（比 localStorage 容量大，适合存题库/对局）。
- 离线：**PWA + Service Worker**（缓存资源，断网可用）。
- 部署：GitHub Pages / CloudStudio，一条命令推送即上线。

**参考开源库（避免重复造轮子）**
- 棋盘/规则：`goban`、`boardmatcher` 等（选型时对比体积与协议）。
- SGF：`@sabaki/sgf` 等标准解析器。
- KataGo WASM：`katago-web` 思路 / ONNX-WASM 推理封装。

```
│  │  └─ katago.js       # KataGo WASM 封装（懒加载）
│  ├─ tsumego/           # 死活题引擎：SGF 解析 + 判定 + 进度
│  ├─ store/             # IndexedDB 封装（对局/进度/错题）
│  └─ ui/                # 页面组件、主题
├─ public/
│  ├─ problems/          # 内置精选死活题（SGF/JSON）
│  └─ katago/            # KataGo WASM + 模型（懒加载资源）
├─ manifest.webmanifest  # PWA
└─ vite.config.js
```

## 7. 关键风险与权衡

- **KataGo WASM 体积大**：首次加载慢，用懒加载 + 加载进度提示缓解；低端机推理可能偏慢，限制线程数/算力。
- **轻量 AI 强度有限**：只适合入门和让子陪练，别指望它多强——这是"免费即时"的代价。
- **规则引擎正确性**：打劫（劫争）、自杀手、数子规则需严谨实现，建议用成熟库或充分测试。
- **题库版权**：导入开源题时注意许可证；内置精选题优先用公版/自编。
- **浏览器存储上限**：IndexedDB 容量足够，但换设备不同步（纯本地设计，刻意不做云端）。
